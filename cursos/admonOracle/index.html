Qué beneficios te da estudiar el curso Administrar y Supervisar Bases de Datos Oracle

  • Categorizar, etiquetar y cumplir con muchas funciones de formato que hacen que su análisis sea mucho más sencillo.

Contexto

Dentro del contexto aprenderás lo esencial para poder desarrollarte dentro de actividades de un administrador de base de datos, así como entender lo que es VirtualBox y máquinas virtuales.

¿Por qué Contexto?

El contexto son conceptos que te ayudara a ser capaz de resolver problemas y asegurar el buen funcionamiento dentro de una aplicación en relación a la base de datos.

¿Para qué sirve el Contexto?

  • Administrar y dar mantenimiento a las bases de datos.
  • Crear máquinas virtuales con la ayuda de una aplicación.
  • Puedes probar otros sistemas operativos.

Características esenciales

  • Proporcionar seguridad a la información sensible que se almacena en las bases de datos.
  • Se pueden ejecutar múltiples sistemas operativos en un solo dispositivo.
  • Se pueden emular casi todos los sistemas operativos, como Linux, Windows, Android, entre otros.

Instalación y configuración del servidor de base de datos

Aprenderás a instalar y configura la plataforma Linux para un servidor de base de datos Oracle, así como los comandos más utilizados de Linux para Oracle.

¿Por qué instalación y configuración del servidor de base de datos?

La instalación y configuración del servidor de base de datos es un procedimiento para instalar el sistema operativo Oracle Linux.

¿Para qué sirve la instalación y configuración del servidor de base de datos?

  • Serás capaz de crear soluciones a través de un entorno virtual basado en la nube.
  • Se pueden realizar análisis de datos.
  • Se puede proporcionar mayor seguridad a la información almacenada en las bases de datos.

Características esenciales

  • Se actualiza todos los días con parches de seguridad.
  • Es un sistema de código abierto.
  • Es capaz de correr en cualquier aplicación Linux.

Crear una base de datos

Las bases de datos son de lo más utilizado en el entorno empresarial, por ello es importante saber crear una base de datos, pues con ello podrás almacenar y gestionar gran cantidad de información.

¿Por qué creación de una base de datos?

La creación de base de datos es la manera de almacenar una gran cantidad de información dentro de una base de datos.

¿Para qué sirve la creación de una base de datos?

  • Se puede organizar información.
  • Se puede gestionar y actualizar información.
  • Crear una base de datos que se adecue a tus necesidades.

Características esenciales

  • Los datos almacenados tiene relación entre sí.
  • Es posible tener una o más tablas que guarden la información.
  • Uno o varios usuarios pueden acceder a la información.

Administrar los estados de la instancia de una base de datos Oracle

Es importante administrar los estados de la instancia de una base de datos Oracle, pues con ello podrás ejecutar algunas acciones que dependen directamente del estado en el que se encuentran tus bases de datos.

¿Por qué administración de los estados de la instancia de una base de datos Oracle??

La administración de los estados de la instancia de una base de datos Oracle es la manera de saber si la base de datos Oracle está activa, montada, desmontada, o en cualquier otro estado.

¿Para qué sirve la administración de los estados de la instancia de una base de datos Oracle?

  • Se puede comprobar la disponibilidad del sistema.
  • Es posible cerrar o desmontar una base de datos.
  • Puedes realizar tareas como administrador y recuperar bases de datos.

Características esenciales

  • Se deben utilizar comandos de acuerdo al estado en el que se desee estar dentro de la base de datos.
  • Las bases de datos pueden encontrarse en cuatro estados diferentes (SHUTDOWN, NOMOUNT, MOUNT, OPEN).
  • Se puede verificar el estado de cada instancia.

Supervisión del uso de espacio

Con la supervisión del uso de espacio es posible obtener métricas del sistema de un espacio de trabajo dentro de la integración de datos y en la ejecución de la información para aplicaciones dentro del espacio de trabajo.

¿Por qué supervisión del uso de espacio?

Proporciona información acerca del resumen sobre el tiempo de las actividades y el uso de estas, también se puede obtener información general de las ejecuciones iniciadas desde aplicaciones de un espacio de trabajo.

¿Para qué sirve la supervisión del uso de espacio?

  • Permite garantizar un mejor rendimiento y disponibilidad.
  • En el proceso de supervisión de una base de datos se comprueba la disponibilidad de espacio libre.
  • Busca posibles errores de base de datos y la conexión con servidores.

Características esenciales

  • Se puede parar e iniciar con una petición.
  • La información mostrada se proporciona en dos secciones.
  • Muestra una sección de resumen.

Administración de memoria

La administración de memoria es capaz de modificarla dinámicamente, mientras la instancia está operando, es decir, es posible aumentar o disminuir su valor sin tener que parar la base de datos.

¿Por qué administración de memoria?

Al crear una base de datos, se pueden establecer nuevos parámetros los cuales tienen casi todos los ajustes de la memoria.

¿Para qué sirve la administración de memoria?

  • Es posible liberar recursos de la memoria.
  • Con la administración de memoria se puede obtener mayor seguridad en una base de datos.
  • Puedes almacenar más cantidad de información sin saturar la memoria.

Características esenciales

  • El espacio de intercambio de memoria, tiene como mínimo tres o cuatro veces el tamaño de memoria física.
  • Ofrece tres formas de gestionar la memoria (Automáticamente, Manualmente, Combinación de automático y manual).
  • Puedes administrar la memoria, mientras la base de datos sigue en funcionamiento.

Monitoreando la base de datos

Se puede tener información detallada y actualizada sobre cómo funcionan las aplicaciones de la base de datos y cuándo se detecta un problema. Para poder identificar problemas antes de que se conviertan fallas.

¿Por qué monitoreo de bases de datos?

Es el proceso para poder monitorear bases de datos los cuales sirven para identificar “cuellos de botella” en el rendimiento así proporcionando medidas para mejorar el rendimiento general.

¿Para qué sirve el monitoreo de bases de datos?

  • Ayuda a identificar y abordar los problemas potenciales en la base de datos.
  • Ayuda a evitar cualquier tiempo de inactividad que pueda experimentar la base de datos.

Características esenciales

  • Es posible identificar problemas de rendimiento, antes de que se conviertan en un problema mayor y causen daños.
  • Se puede administrar cualquier problema de rendimiento.

Administrar y supervisar transacciones

Una transacción es una única operación de configuración, por ejemplo, cuando se crea un nuevo usuario. Para asegurar que dicha transacción se complete cuando los recursos no están disponibles, se escriben en el almacén persistente de transacciones donde se administran.

Es posible buscar transacciones en el almacén persistente de transacciones para ver el estado en que se encuentran.

¿Por qué administrar y supervisar transacciones?

Mediante un administrador gráfico, la persona encargada es capaz de controlar el momento en que se debe mantener la transacción para poder desde esa herramienta administrar y supervisar las mismas.

¿Para qué sirve el monitoreo de bases de datos?

  • Habilita procesamiento asíncrono.
  • Mantiene las transacciones.
  • Busca transacciones que se encuentran en cualquier estado.

Características esenciales

  • Supervisa métricas clave de rendimiento y configuración.
  • Utiliza el hub de rendimiento para diagnosticar rápidamente problemas de rendimiento.

Administrar los archivos que controlan la operación de base de datos

Un archivo de datos contiene todos los datos de tu base de datos, a su vez estos datos se almacenan físicamente en archivos de datos en la base.

¿Por qué administrar los archivos que controlan la operación de base de datos?

Son archivos que almacenan los datos en sí de la base de datos. Estos graban la información de las tablas.

Para lograr una optimización, su funcionamiento y su gestión, Oracle utiliza una estructura que puede relacionar la lógica de la base de datos y a su vez la parte física.

¿Para qué sirve administrar los archivos que controlan la operación de base de datos?

  • Los archivos forman estructuras y estas a su vez forman un esquema en la base de datos.
  • Es posible relacionar la parte lógica de la base de datos con la parte física.

Características esenciales

  • Es la estructura lógica de una base de datos.
  • Oracle llama a las estructuras Logical Storage Structures, por lo que en español se suele traducir como estructuras lógicas de almacenamiento.

Administrar tablas

Las tablas son parte fundamental en una base de datos, una tabla está compuesta de filas y columnas.

¿Por qué administrar tablas?

Para guardar los datos, por ejemplo de un cliente se deben crear tablas las cuales a su vez se tienen que administrar pues dichos valores serán las filas y columnas de nuestra tabla.

¿Para qué sirve administrar tablas?

  • Para guardar datos.
  • Para modificar datos.
  • Para eliminar datos.

Características esenciales

    Se tienen que analizar varias cosas, como:

  • Nombre de la tabla.
  • Tipo de dato de cada columna.
  • Bajo que esquema la vamos a crear.

Administrar índices

Los índices son estructuras asociadas a tablas, una tabla que almacena los campos indexados y se crean para acelerar las consultas.

¿Por qué administrar índices?

Estos se emplean para facilitar la obtención de información de una tabla, pues funciona igual que el índice de un libro, ya que permite } encontrar datos rápidamente y localiza registros dentro de una tabla.

¿Para qué sirve administrar índices?

  • Puedes acceder a los registros de una tabla rápidamente.
  • Es posible localizar datos específicos rápidamente.
  • Se mantiene un mayor orden en las bases de datos.

Características esenciales

  • Recorren las tablas.
  • Emplean índices para recorrer la estructura.
  • Posibilita accesos directos y rápidos.

Particionamiento

Es una de las funcionalidades más importantes de la base de datos Oracle la cual mejora el desempeño, la capacidad de administración y la disponibilidad de miles de aplicaciones.

¿Por qué particionamiento??

El particionamiento ayuda a mejorar el desempeño de ciertas operaciones de mantenimiento y consultas. También con el particionamiento se puede reducir el costo total de propiedad de los datos.

¿Para qué sirve el particionamiento??

  • Permite subdividir una tabla.
  • Cada partición tiene su propio nombre y sus propias características de almacenamiento.
  • Amplia flexibilidad a la hora de administrar ese objeto.

Características esenciales

  • Los objetos (tablas e índices) se particionan a partir de una serie de columnas.
  • Se puede administrar por diferentes niveles.
  • Las tablas e índices se pueden particionar en unidades más pequeñas y administrables.
